The Webex CLI includes a sample [Claude Code](https://claude.com/claude-code) skill that enables AI agents to query and manage your Webex environment.

## Setup

1. Install the Webex CLI (see [Installation](/webex-cli/installation/))
2. Login: `webex login`
3. Copy or symlink the skill into your Claude Code skills directory:

```bash
# Option A: Symlink (recommended — stays in sync with updates)
ln -s /path/to/webex-cli/skill ~/.claude/skills/webex-cli

# Option B: Copy
cp -r /path/to/webex-cli/skill ~/.claude/skills/webex-cli
```

4. If the `webex` binary is not in your `$PATH`, edit `skill/SKILL.md` and update the binary path.

## What the Skill Can Do

Once configured, you can ask Claude Code to:

- **List resources**: "List all CC sites", "Show me the call queues"
- **Inspect configuration**: "What teams are on the Cloverhound WxCC site?"
- **Debug issues**: "Why can't agents log in?", "Check the business hours schedule"
- **Analyze the environment**: "Give me an overview of the contact center setup"
- **Run commands**: "Get the details for entry point X"

## Example Prompts

```
/webex-cli list all CC entry points and show which ones are chat vs voice

/webex-cli how many agents are assigned to each team?

/webex-cli check auth status and list sites
```

## Customization

The skill file at `skill/SKILL.md` can be customized:

- Add your org-specific context (site names, team structures)
- Restrict which commands the skill can use
- Add common workflows as examples

See the [Claude Code skills documentation](https://docs.anthropic.com/en/docs/claude-code/skills) for more on building skills.
